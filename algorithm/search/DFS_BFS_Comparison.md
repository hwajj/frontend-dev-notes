

# ✨ DFS vs BFS — 깔끔 버전 (중복 제거 완료)

## 1. 📊 핵심 차이점 (이 표 하나면 사실 절반 끝남)

| 구분    | DFS         | BFS        |
| ----- | ----------- | ---------- |
| 자료구조  | 스택 / 재귀     | 큐          |
| 탐색 방식 | 깊게 끝까지      | 가까운 것부터    |
| 최단 거리 | ❌ 안됨        | ✅ 됨        |
| 메모리   | 상대적으로 적음    | 상대적으로 많음   |
| 시간복잡도 | O(V+E)      | O(V+E)     |
| 비유    | 미로에서 한쪽으로 쭉 | 한 칸씩 주변 탐색 |

---

## 2. 🎯 언제 무엇을 쓸까? (중복 제거 후 진짜 기준만)

### ✅ DFS를 쓰는 경우 (백트래킹/전체 탐색 계열)

* **모든 경우의 수 탐색**

  * 조합/순열/Subsets
  * N-Queen, Sudoku
* **연결된 영역 찾기 (DFS 전통 강자)**

  * Number of Islands, Max Area of Island
* **최단 경로 필요 없음**

  * Binary Tree Paths

### ✅ BFS를 쓰는 경우 (최단 거리/레벨 탐색 계열)

* **최단 거리/최소 단계**

  * Word Ladder
  * 미로 최단 거리
* **레벨 단위 탐색**

  * Level Order Traversal
* **가중치 없는 그래프 최단 경로**

  * 네트워크 최단 경로

### 🔄 둘 다 가능한 경우

* 연결된 영역 탐색 (DFS/BFS 상관 없음)
* 그래프 방문하여 도달 가능 여부 확인

---

## 3. 📋 문제 유형별 선택 가이드 (중복 요소 삭제 후 축약)

### BFS가 잘 맞는 문제

* “최단 거리”, “몇 단계 만에” → 무조건 BFS
* 레벨 단위 문제 → BFS가 편함

### DFS가 잘 맞는 문제

* “모든 조합 구해라”
* “백트래킹으로 조건 맞춰라”
* “영역 크기 구해라”

---

## 4. 🔍 빠른 선택 기준 (그냥 이 표보다 이게 더 직관적)

```
최단 거리 필요?
  YES → BFS

모든 경우 다 봐야함?
  YES → DFS

레벨별 구해야함?
  YES → BFS

연결된 영역?
  DFS/BFS 둘 다 가능

메모리 아껴야 함?
  DFS 선호
```

---

## 5. 💡 실전 팁 (중복 줄이고 핵심만)

* DFS → 백트래킹에 최적, 불필요한 루트 가지치기 가능
* BFS → 최단 거리 찾으면 바로 종료되어 효율적
* DFS는 코드 짧고 구현 편하고, BFS는 큐 관리 필요
