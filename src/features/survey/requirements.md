# 설문지 시스템 요구사항

## 🎯 시스템 개요
사용자가 설문지를 생성하고 관리할 수 있는 웹 애플리케이션

## 📋 주요 기능 요구사항

### 1. 설문지 생성 및 편집
- [x] 새 설문지 생성
- [x] 설문지 제목 수정
- [x] 질문 추가/수정/삭제
- [x] 질문 유형 선택 (텍스트, 객관식, 평점)

### 2. 데이터 관리
- [x] 임시 저장 (draft)
- [x] 최종 확정 (committed)
- [x] 자동 저장
- [x] 데이터 복구

### 3. 사용자 경험
- [x] 실시간 미리보기
- [x] 직관적인 UI/UX
- [x] 반응형 디자인
- [x] 에러 처리 및 피드백

## 🔍 케이스 10개

### 케이스 1: 새 설문지 생성
**시나리오**: 사용자가 처음 접속하여 새 설문지를 생성
**입력**: "새 설문지" 버튼 클릭
**예상 결과**: 빈 설문지가 생성되고 편집 모드 진입

### 케이스 2: 설문지 제목 입력
**시나리오**: 사용자가 설문지 제목을 입력
**입력**: "고객 만족도 조사" 텍스트 입력
**예상 결과**: 제목이 실시간으로 업데이트되고 저장

### 케이스 3: 텍스트 질문 추가
**시나리오**: 사용자가 텍스트 입력 질문을 추가
**입력**: "귀하의 이름을 입력해주세요" 텍스트 + 텍스트 타입 선택
**예상 결과**: 질문이 목록에 추가되고 자동 저장

### 케이스 4: 객관식 질문 추가
**시나리오**: 사용자가 객관식 질문을 추가
**입력**: "선호하는 색상을 선택해주세요" + 객관식 타입 선택
**예상 결과**: 객관식 질문이 추가되고 옵션 입력 필드 표시

### 케이스 5: 평점 질문 추가
**시나리오**: 사용자가 평점 질문을 추가
**입력**: "서비스 품질을 평가해주세요" + 평점 타입 선택
**예상 결과**: 평점 질문이 추가되고 1-5점 스케일 표시

### 케이스 6: 질문 수정
**시나리오**: 사용자가 기존 질문을 수정
**입력**: 질문 텍스트를 "귀하의 연령대를 선택해주세요"로 변경
**예상 결과**: 질문이 실시간으로 업데이트되고 저장

### 케이스 7: 질문 삭제
**시나리오**: 사용자가 불필요한 질문을 삭제
**입력**: 특정 질문의 삭제 버튼 클릭
**예상 결과**: 질문이 목록에서 제거되고 저장

### 케이스 8: 임시 저장
**시나리오**: 사용자가 작업 중인 설문지를 임시 저장
**입력**: 자동 저장 또는 수동 저장
**예상 결과**: 현재 상태가 draft로 저장되고 브라우저 새로고침 시 복구

### 케이스 9: 설문지 확정
**시나리오**: 사용자가 완성된 설문지를 최종 확정
**입력**: "설문지 확정" 버튼 클릭
**예상 결과**: 설문지가 committed 상태로 변경되고 draft 삭제

### 케이스 10: 에러 처리
**시나리오**: 네트워크 오류나 저장 실패 상황
**입력**: 저장 중 네트워크 끊김
**예상 결과**: 적절한 에러 메시지 표시 및 재시도 옵션 제공

## 🚀 비기능적 요구사항

### 성능
- 페이지 로딩 시간 < 2초
- 질문 추가/수정 응답 시간 < 500ms
- 자동 저장 간격: 5초

### 호환성
- Chrome 90+, Firefox 88+, Safari 14+
- 모바일 반응형 지원
- PWA 지원

### 보안
- XSS 방지
- CSRF 토큰 사용
- 입력 데이터 검증

### 접근성
- WCAG 2.1 AA 준수
- 키보드 네비게이션 지원
- 스크린 리더 호환
